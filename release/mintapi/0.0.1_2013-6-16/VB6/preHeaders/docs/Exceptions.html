Description:
Exception library is provided to throw exception same as .net framework to ease the error managing.

This provides a syntax known as throw to raise your exception , an exception can also be operating system exception and when raise your application will be terminated by the operating system.

This allows you to throw an error without write any error string or description about your error.
All you need for throwing an error is to write following code in your statements:

throw [Exception Type]
throw NotImplementedException
throw SystemCallFailureException
  .
  .
  .

Also can specify the exception information using throw0 syntax:

throw0("My Application/Dll/Libray/...","modMain/frmMain/...","ArcSin/Main/...","Some Description About Exception","Arguments x As Int32",[Error Number],__,__,__)

throwT method throw an exception with two parameters Description of the exception and Location ,the location of the exception occurance.
throwT("Item does not exists.","mydll.myclass.mymethod()")

throwN function provided to throw an exception without any information about it ,this used for automation actions.

Location() creates a record to store passed arguments to it and return it as string to use in exception creators.

Exception Creators
------------------
-SystemException() will create an exception that will throw an error to the operating system kernel ,operating system then terminates the application.
-Exception() default exception creator method.
-############Exception() creates specified exception to be throwed.
ex: NotImplementedException() | usage: throw NotImplementedException

CountAllExceptions() returns all throwed exceptions including system exceptions.
ClearAllExceptions() clear all counted exceptions.
